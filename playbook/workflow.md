# 内容生产工作流 v2

**Updated: 2026-04-08**

---

## 核心原则

**每篇内容 = 时效性事件 × 独特视角**

没有时效锚点的观点是软文。没有独特视角的搬运是营销号。两者结合才是高智感内容。

## Phase 1：选题（Zico 5 min + Claude 辅助）

### 触发条件（满足任一即可启动）

1. **热点事件**：某个 KOL 说了一句引发争论的话 / 某个开源项目突然爆火 / 某家公司发布重大更新
2. **概念碰撞**：Zico 在日常工作/思考中产生了一个洞察，恰好能接到某个近期事件上
3. **信息差搬运**：英文圈已经在讨论但中文圈还没跟上的话题

### 选题评估标准

| 维度 | 问题 | 最低标准 |
|---|---|---|
| 时效性 | 这件事发生在多久以内？ | 7 天内最佳，30 天内可接受 |
| 信息增量 | 中文社交媒体上有人说过这个角度吗？ | 要么没人说，要么说得不够好 |
| 共鸣度 | 目标读者看了会想转发/评论吗？ | 至少能引发"我也观察到了"的共鸣 |
| Zico 适配度 | 这件事和 Zico 的认知领域有交集吗？ | 必须能接到 5 个主题柱之一 |

## Phase 2：内容生产（Claude 为主，Zico 审核）

### 内容类型矩阵

| 类型 | 描述 | Zico 投入 | 示例 |
|---|---|---|---|
| **A. 观点评论** | 时效事件 + Zico 独特解读 | 核心观点 3-5 句 | "Karpathy 说了XX，我的理解是..." |
| **B. 概念命名** | 提炼一个现象并命名 | 观察 + 命名 | "容错率套利" |
| **C. 信息差搬运** | 英文圈热议 → 中文首发解读 | 选题 + 审核 | YC 最新博客解读 |
| **D. 趋势拆解** | 多个信号归纳出一个趋势 | 方向判断 | 多个 Agent 项目 → Agent 经济 |
| **E. 实践分享** | 自己的真实工作经验（匿名化） | 真实经历 | "我在产品后台发现..." |

### 生产步骤

```
1. Claude 抓取时效性素材（搜索最新动态）
2. Zico 给出核心视角 / Claude 提议视角供 Zico 选择
3. Claude 写初稿（遵循 voice-style-guide.md）
4. Zico 审核修改（重点：真实感、个人痕迹、品味）
5. Claude 生成封面方案 + 配图文案 + 发布策略
6. 发布
```

### 关键约束

- 初稿中必须包含至少一个**可追溯的时效性锚点**（谁说了什么、什么项目火了、什么数据出来了）
- 锚点不用具名，但要具体到读者能去验证的程度
- 观点可以是阐释他人的观点，不必都是原创论点
- 但必须有 Zico 自己的**一句话判断**（哪怕是"我同意/不同意/觉得他漏掉了一个角度"）

## Phase 3：平台适配 + 发布

### 小红书版本

- 字数：600-1200
- 开头即锚点（"XX最近说了一句话..."/"一个开源项目这周突然上了GitHub trending..."）
- 结构：事件 → 解读 → 延伸问题 → 互动引导
- 封面：延续统一视觉风格

### 后续扩展（暂不启动，节奏稳定后开）

- 即刻：短版本，1-3 段
- 知乎：长版本，加论据
- 公众号：完整版，加背景铺垫
