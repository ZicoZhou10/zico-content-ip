# Today's Candidates

## Candidate 1: 纽约州成全美首个州级AI算力禁令——Hochul 叫停超大规模数据中心新建许可，39个待审项目一夜冻结（July 14）

- **Event**: 2026年7月14日，纽约州长 **Kathy Hochul** 签署行政令，推出全美首个针对超大规模数据中心的州级暂停令（moratorium）：**所有需要50兆瓦以上电力的新建超大规模数据中心，最长暂停1年的州级环保许可审批**，以建立"保护电费用户、环境和能源电网的监管框架"。触发数据：纽约州目前有4座超大规模数据中心已运营，另有**39个待审申请**和25个在NYISO（纽约独立系统运营商）互联队列中的提案（合计约9,340兆瓦待建产能）。Hochul 原话："这些超大规模AI数据中心消耗的电力是惊人的，正在实质性地威胁超越电网承载上限，同时推高本地用户的电费。"能源背景：纽约住宅电价自2019年以来累计上涨近 **68%**，被明确归因于数据中心电力需求激增。新要求包括：运营商须支付能源溢价或自建发电设施，本地社区须从数据中心项目中获得基础设施利益补偿。来源：CNBC（2026年7月14日）、Washington Post（2026年7月14日）、The Hill（2026年7月14日）、纽约州长官网行政令全文。
- **Source**: https://www.cnbc.com/2026/07/14/new-york-ai-data-center-ban.html | https://www.washingtonpost.com/technology/2026/07/14/new-york-becomes-first-state-impose-data-center-moratorium/ | https://www.governor.ny.gov/news/first-statewide-moratorium-new-hyperscale-data-centers-launched-governor-kathy-hochul | https://thehill.com/policy/technology/5966562-new-york-ai-data-center-moratorium/
- **Timeliness**: 2天前（2026年7月14日，CNBC / Washington Post / The Hill 多家一线媒体同日报道）
- **Topic pillar**: 深层AI思考
- **Zico's angle**: 这件事和过去所有AI监管都不一样。

  过去几年的AI治理，所有行动都在三个层次展开：**内容层**（GDPR、COPPA，管数据）、**模型层**（EU AI Act、NIST框架，管算法行为）、**应用层**（中国拟人化AI新规，管交互形式）。7月14日发生的是第四种：**基础设施层监管**——不是管AI能做什么，而是管AI能用多少电。

  这个区别很重要，因为它触及的不是AI的能力边界，而是AI的物理边界。Hochul 的行政令本质上是一个能源分配命令：纽约说，我的电网承受不了你们的增长速度，先停一年。

  从数据中心的角度理解：39个待审申请瞬间冻结。这39个项目代表的是未来的算力——无论是 Nvidia 的推理集群、OpenAI 的训练基础设施还是企业自建的 Agent 后端。现在这些算力的地理坐标需要重新规划：不能建在纽约了，那么会建在哪里？德克萨斯？弗吉尼亚？还是出境去欧洲或亚洲？

  这是一个没有被充分讨论的结构性问题：**AI 治理的新战场是能源地图，而不是法律条文**。谁能控制足够多的清洁电力和冷却水资源，谁就能建造更多算力。治理手段的切换，从"你不能做什么"变成了"你能用多少瓦"。

  对 Zico 的"AI 作为新物种"框架：一个物种的扩张最终受到生态位的物理限制。AI 的生态位是算力，算力的生态位是能源，能源的生态位是电网。2026年7月14日，第一张能源边界被划定了。

- **Resonance hook**: 7月14日，纽约州长 Kathy Hochul 签署行政令：全美第一个州级超大规模AI数据中心建设禁令。理由不是安全，不是隐私，是电——AI数据中心的用电需求正在"实质性地威胁超越电网承载上限"，纽约住宅电价5年已涨68%。39个待审数据中心申请瞬间冻结。这不是在管AI能做什么，是在管AI能用多少电。治理战场的坐标，从法律转移到了能源地图。
- **Recommended priority**: high

## Candidate 2: SemiAnalysis：Claude Code 贡献全球 GitHub 提交量逾7%——Anthropic ARR 六个月从90亿增至600亿，API毛利率超80%（July ~10）

- **Event**: 2026年7月（约7月10日），**SemiAnalysis**（Dylan Patel 团队主导的AI行业深度分析通讯）发布报告《Anthropic 3Q26 Profit Over $1B: The Anthropic IPO Financials Sneak Peak》，披露 Anthropic 关键财务和产品数据：（1）**Claude Code** 现已贡献**全球 GitHub 所有提交量的逾7%**，Monthly NNARR（月净新增年化经常性收入）从2026年1月的30亿美元跃升至3月的110亿美元，主要驱动力即 Claude Code；（2）**ARR（年化经常性收入）** 从2025年底的**90亿美元**在约6个月内增长至**600亿美元**（7倍增长）；（3）**API 业务毛利率超过80%**，整体毛利率从2024年的 **-94%**（深度亏损）回升至2026年的 **+60%** 区间；（4）Q3 2026 GAAP EBIT 预计突破**10亿美元**；（5）ARR 构成：75-85% 来自 API 用量计费，消费者订阅仅占约5%。背景：Anthropic 已于2026年6月1日向 SEC 机密提交 IPO 申请，估值约9650亿美元。
- **Source**: https://newsletter.semianalysis.com/p/anthropic-3q26-profit-over-1b-the | https://www.htx.com/news/semianalysis-anthropics-q3-profit-to-exceed-1-billion-wr7NEMS1/ | https://www.tradingkey.com/analysis/stocks/us-stocks/262018103-semianalysis-anthropic-opanai-ai-ipo-tradingkey | https://yellow.com/news/anthropic-q3-2026-profit-1b-ipo-filing
- **Timeliness**: 约6天前（SemiAnalysis 约2026年7月10日发布；HTX Insights、TradingKey、ChainCatcher 随后报道）
- **Topic pillar**: AI协作实践
- **Zico's angle**: 7% 这个数字是一个阈值，不是一个指标。

  先把它还原成具体尺度：GitHub 每天约有数百万次提交。7% 意味着每14次 commit 里，有1次是 Claude Code 写的。这不是用户"借助AI辅助写代码"——这是 Claude Code 已经成为全球软件生产的一个结构性参与者。

  这个阈值有一个关键含义：**你现在在 GitHub 上看到的代码，很难判断哪些是人写的，哪些是 Claude Code 写的**。不是因为 AI 代码质量不好，而是因为它通过了 code review、通过了测试、被合并进了主分支。代码库的"DNA"正在发生改变，而这个改变在大多数团队的 git log 里是不可见的。

  SemiAnalysis 的财务数据提供了另一个角度：Claude Code 是 Anthropic 从亏损到盈利的直接原因——不是 Claude 聊天，不是 Claude 企业版，而是 Claude Code。毛利率从 -94% 到 +60% 的转变，发生在 Claude Code 爆发的这几个月。这意味着 **"AI 协作工具"不只是一个产品品类，它是 AI 公司的利润结构根基**。

  下一个值得问的问题：被 AI 贡献7% 的"提交"之后是什么？软件开发的认知分工会如何重组？如果 Claude Code 写了7% 的代码，谁在做软件架构判断，谁在理解这些代码真正做了什么？

  中文媒体会报"Anthropic 即将上市，估值近万亿"，不会分析：**驱动 Anthropic 从亏损到盈利的核心产品，是一个贡献了全球7% GitHub提交的AI协作者——不是聊天机器人，是写代码的同事。**

- **Resonance hook**: SemiAnalysis 近日披露：Claude Code 已贡献超过全球 GitHub 提交量的7%——每14次 commit，有1次是 Claude Code 写的。同一份报告：Anthropic ARR 从90亿增至600亿（6个月，7倍），API毛利率超80%，整体毛利率从2024年的 -94% 升至 +60%。这个转变的时间节点和 Claude Code 爆发完全重合。驱动 Anthropic 盈利的，不是更好的聊天，是那个写代码的同事。
- **Recommended priority**: high

## Candidate 3: 智谱创始人唐杰发内部信力挺AI开放——三天前路透报道北京正在讨论限制中国AI模型向海外输出，DeepSeek 和 Qwen 首当其冲（July 12）

- **Event**: 2026年7月12日，**Bloomberg** 独家披露：**智谱AI（Zhipu AI）** 创始人兼首席科学家**唐杰（Tang Jie）**向全体员工发出内部信《巨浪已至》（"The Giant Wave Has Arrived"），旗帜鲜明表态：**前沿人工智能必须保持向所有人开放**，强调"真正的安全来自于广泛参与、共享和监督，而非技术壁垒"。这封信是对5天前（7月7日）路透社报道的直接回应：路透报道称，**中国商务部**召集**阿里巴巴、字节跳动和智谱AI**高管开会，讨论是否对中国最先进AI模型实施**海外输出管控**——涵盖尚未发布的模型，以及开放权重（open-weight）模型（即类似 **DeepSeek** 的开源发布形式）；同时讨论对泄露AI技术者的国家安全法制裁，以及对境内AI创业公司融资来源的新限制。受波及模型包括 DeepSeek、Qwen（通义千问）和 GLM（智谱），即同日（7月12日）Goldman Sachs 向西方企业推荐采购的三款中国AI模型。
- **Source**: https://www.bloomberg.com/news/articles/2026-07-12/zhipu-s-chinese-founder-says-frontier-ai-should-stay-open-to-all | https://thenextweb.com/news/zhipu-tang-jie-frontier-ai-open-to-all | https://www.explainx.ai/blog/china-overseas-ai-model-restrictions-reuters-july-2026 | https://cryptobriefing.com/china-tightens-control-domestic-ai-technology/ | https://letsdatascience.com/news/china-considers-restricting-overseas-access-to-advanced-ai-m-812db030
- **Timeliness**: 4天前（2026年7月12日，Bloomberg 独家；Reuters 7月7日首发限制讨论报道）
- **Topic pillar**: AI产品战略
- **Zico's angle**: 7月12日这一天发生了三件事，需要放在一起读：Goldman Sachs 发报告推荐西方企业采购中国AI模型（DeepSeek、Qwen、GLM），唐杰发内部信力挺AI开放，中国商务部正在讨论是否限制这些模型向海外输出。

  Goldman Sachs 的报告解决的是"用不用"的问题——给了西方企业一个机构背书。唐杰的内部信解决的是"能不能保持开放"的问题——说明这件事在中国AI行业内部存在真实的分歧。中国商务部的会议是一个悬而未决的问题——中国政府最终是否会关上这扇门。

  三件事同时发生，构成了一个完整的决策场景：**西方企业刚刚获得了一份来自华尔街的采购许可，发现下游供应链上挂着一把尚未落下的锁。**

  先把技术后果说清楚。如果中国限制开放权重模型的海外输出，DeepSeek 和 Qwen 的开源版本就不再自由流通——意味着运行在 AWS Bedrock 和 Google Vertex 上的中国模型服务可能随时面临合规风险，意味着基于中国开源模型训练的微调版本的法律地位变得不确定，意味着 Goldman Sachs 报告里推荐的那条"高吞吐低成本任务交给中国模型"的路径，在供应链层面存在主权变量。

  唐杰内部信的信号强度值得单独说。当一个创始人需要写"我们坚定支持开放"给员工，通常意味着"关闭"已经是一个被认真讨论的选项。分歧一旦以内部信的形式外泄，就说明行业内部对政府限制方向已经有了预期——不是是否会发生，而是何时会发生。

  对 Zico 在做AI产品的视角：做产品时，供应链风险的评估比技术评估更重要。如果你今天正在把 DeepSeek 集成进你的 Agent 产品，你现在需要考虑的不是它的性能，是它的主权风险。

- **Resonance hook**: 7月12日，Bloomberg：智谱创始人唐杰向员工发内部信《巨浪已至》，表态前沿AI必须对所有人保持开放。同一天，Goldman Sachs 向西方企业推荐中国AI模型（DeepSeek、Qwen、GLM）。5天前，路透：中国商务部召集阿里、字节、智谱开会，讨论是否限制中国最先进AI模型向海外输出——包括 DeepSeek 的开放权重版本。Goldman Sachs 刚给了西方企业一张采购许可，下游供应链上挂着一把尚未落下的锁。如果中国关上这扇门，"用中国模型做高吞吐 Agent 任务"这条路的风险图谱会瞬间改变。
- **Recommended priority**: high
