为什么是 Olah？因为 Olah 是 AI 领域里最接近回答"我们能理解 AI 内部在做什么吗"这个问题的研究者。他的工作——追踪神经网络里的"特征"和"电路"，理解哪个神经元激活了哪个概念，模型内部的表征是如何组织的——是在做一件非常艰难的事：把一个黑盒变成可以被人类检查的系统。

这意味着，梵蒂冈与 Anthropic 的合作，不是一次 PR 事件，而是一次关于"AI 安全的底层逻辑是什么"的隐含声明：你不能仅仅通过政策治理你不理解的系统。可解释性——科学意义上的理解 AI 内部——才是所有监管框架的前提。

如果 Olah 的研究方向是对的，"AI 可解释性"将成为 2026-2030 年最重要的基础研究领域。不是因为它会让 AI 更聪明，而是因为它是人类保持对 AI 真实掌控感的唯一路径。

## Candidate 2: Boston Dynamics Atlas 学会扛 110 磅，现代汽车计划工厂部署 25,000 台——韩国工会：没合同一台都不让进（May 18-22）

值得分析的是**这道经济学题**：Hyundai 为什么要在还没确定具体工位的情况下，先建一个年产 30,000 台机器人的工厂？

答案不在技术，在赌注结构。Hyundai 建设这个工厂（$26 亿美元 U.S. 投资的一部分），是在押注：**到 2028 年，人形机器人的单位经济会成立**。不是"可能成立"，而是"必须成立"——因为工厂一旦建成，30,000 台/年的产能摆在那里，如果没有足够的订单，资本就浪费了。

现在已知的数字：$145,000/台，24/7 运转，两年成本 ≤ 1 个工人年薪 × 2。这个算式看起来成立——但这是在理想条件下的理论值，没有停机维修、没有重新训练、没有导入新工位的调试成本。

韩国金属工会的人比任何人都更仔细地算过这道题。他们的诉求——"引入机器人时保障全员就业"——不是情绪反应，是谈判策略：**如果机器人的经济收益是真实的，工会要分走谈判桌上属于劳动者的那部分**。Kia 工会要求 30% 营业利润作为奖金，直接把机器人收益分配问题从"是否"推进到了"多少"。

这是 Physical AI 规模化的第一个真实压力测试：不是在实验室里，而是在工厂谈判桌上。单位经济、工会合同、量产工厂的资本承诺——这三个因素同时压缩到 2028 年，是具身 AI 走向工业主流之前必须解决的最真实问题。

对 Zico 受众来说，这个案例的价值是：判断 Physical AI 是否到了真正可操作的时间窗口，不用看 benchmark，看 Hyundai 和韩国工会的谈判结果就够了。

## Candidate 3: DeepSeek 招募前 Jane Street 工程师 Cui Tianyi 组建 Code Harness 团队——AI 竞争从"模型之战"转向"工程之战"（May 20）

- **Event**: 2026年5月20日（6天前），DeepSeek 在招聘平台同时发布产品经理和 R&D 工程师职位，岗位说明显示新产品名称为 **"Code Harness"**，岗位地点：北京。同日，DeepSeek 员工 **Deli Chen** 在 X 上发文，确认 **Cui Tianyi**（崔天一）已加入 DeepSeek 并担任 Code Harness 项目的技术负责人（technical principal）。Cui Tianyi 简历：在香港 **Jane Street** 工作近 9 年，主要从事股票（equities）和固收（fixed income）软件开发与研究。Code Harness 的含义：将 DeepSeek V4 模型封装进**确定性执行框架**（deterministic scaffolding）——处理工具调用、错误恢复、外部 API 集成、验证逻辑——让 LLM 可以自主运行复杂的长链任务，而不只是单轮对话。目前无产品上线时间表，预计 6-12 个月后进入产品形态。来源：South China Morning Post（May 20，独家报道）、IndexBox（May 20，雇用细节）、BigGo Finance（May 22，行业分析："From CLAUDE.md to DeepSeek Harness: AI Coding Tools Shift From 'Model Battles' to 'Engineering Battles'"）。

- **Timeliness**: 6 days ago（2026年5月20日）
- **Topic pillar**: Agent 经济
- **Zico's angle**: DeepSeek 招了一个量化金融工程师来做 AI Agent 脚手架。这个选人逻辑说明的问题，比招聘公告本身更有价值。

Jane Street 是这个行业里对"模型 + 执行环境"组合理解最深的机构之一。他们的核心工程文化：一个概率模型单独运行价值有限，**你需要一套决定性的执行层，来保证模型的输出在现实世界里变成可靠的行动**。这套层处理的问题不是模型有多聪明，而是：调用失败怎么重试？结果验证失败怎么回滚？外部 API 的格式不对怎么修正？边界条件怎么硬编码？量化交易领域有 20 年的时间在回答这些问题，最后形成了一套叫做"策略执行基础设施"的工程范式。

Cui Tianyi 带来的不是某个特定算法，而是这整套**工程文化**：把 LLM 当成一个高级概率推理器，然后用一套严格的确定性外壳让它在现实中可靠运行。

这正是当下 AI Agent 领域最缺少的能力。大多数 Agent 框架还在解决"怎么让模型多想两步"的问题，而真正的生产可用性问题是："当 Agent 运行 47 步之后出错，系统能不能恢复？能不能知道从哪里恢复？能不能对错误有可审计的记录？"这些不是模型能力问题，是执行基础设施问题。

DeepSeek 的这个动作告诉我们：**AI 竞争的前沿正在从"谁的模型 benchmark 更高"向"谁的 Agent 执行环境更可靠"迁移**。当 DeepSeek 开始招募量化金融背景的工程师来做这件事，说明他们对下一阶段竞争的判断已经有了方向。

对 Zico 受众——做 AI 产品的人——这里有一个直接可操作的框架参考：**你的 AI 产品是否有一个"harness"？** 不是说有 CLAUDE.md 或者写了 system prompt，而是有没有一套独立于模型的确定性执行逻辑，让你的 Agent 在失败时可以可靠恢复？如果没有，你做的不是 AI 产品，你在做 AI demo。
- **Resonance hook**: 2026年5月20日，DeepSeek 宣布组建 Code Harness 团队。技术负责人是 Cui Tianyi，前 Jane Street 工程师，在香港量化金融做了 9 年。这个团队要做的事是：把 DeepSeek V4 模型装进一套确定性执行框架，让它能自主运行复杂任务而不只是聊天。Jane Street 做 AI 最大的工程贡献之一，就是"模型不够，你还需要一个可靠的执行环境"这件事。DeepSeek 现在要把这个知识带进 LLM。从模型之战到工程之战。
- **Recommended priority**: medium
