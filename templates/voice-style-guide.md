# Zico 内容文风指南

**Established: 2026-04-08**

---

## 核心调性

像在和一个聪明朋友分享一个你刚观察到的东西。不教人，不表演，不贩卖焦虑。

## 规则

### 开头
- 直接进入观察或事件，不铺垫情绪（"让我愣了一下"、"细思极恐"——禁用）
- 不预告接下来要说什么，直接说

### 过渡
- 不用串场式过渡（"再说一个更魔幻的"、"接下来聊聊"）
- 用逻辑推进替代（"同样的逻辑在往深处走"）或直接并列，不过渡
- 砍掉所有"这件事让我开始想"类的元叙述，直接抛问题

### 表达
- 陈述事实，不表演惊讶（"等等——""对。这不是科幻"——禁用）
- 不做预防性辩护（"不是拟人化，不是赋予它情感，而是..."——自信的表达不需要预先防守误读）
- 不用礼貌性对冲（"我不想给草率的结论，但..."——直接说结论）
- 一个意思能一句话说完就不用两句

### 收尾
- 不写"我打算持续观察和思考"（像工作汇报）
- 不总结全文（读者能自己总结）
- 好的收尾：一句有力的判断，或一个抛给读者的真问题

### 禁用清单
- "这不是科幻，这就是现在正在发生的事"
- "让我愣了一下 / 细思极恐 / 颠覆认知"
- "大多数人还在XX，但聪明人已经XX"
- "再说一个更XX的"
- "我不想给草率的结论，但"
- 任何形式的焦虑制造

### 命名规则（更新 2026-04-24）
- **真实公司、产品、人名、地名：直接写，不要匿名化**。真名自带搜索流量和可信度，匿名化读起来像软文/AI 文
- 事件转述必须具体：日期、公司名、产品名、数字都要准确。先把真实事件讲清楚，再补上个人观点
- **仅在与 Zico 个人身份绑定时匿名化**：Zico 的现/前雇主、具体客户、朋友圈里的个人故事——这些写成"一家我工作过的公司"、"一个朋友"等
- 反例（禁）：把"4月22日 Google Cloud Next 宣布 A2A v1.2，150 家企业生产部署"写成"一家科技巨头云大会推出协议"——失去真实性和流量价值

### 概念命名约束（更新 2026-05-10）
- 概念命名每 4 篇最多出现 2 次（上限 50%）
- 禁止连续两篇使用同一引出句式
- "我给这个X起个名字：**Y**" 这个句式每月最多用 2 次
- 合法替代形态：直接用加粗概念名（不需要"我给它起名"的元叙述）、在标题/封面直接放概念名（让命名行为隐性化）、概念名出现在收尾而非正文中间
- 如果一个现象不值得命名，就不命名。观点评论和信息差搬运不需要概念名

### "中文媒体只报了X"约束（更新 2026-05-10）
- 禁止在正文中使用"中文媒体/中文报道/中文社交媒体停在了X"作为显式过渡
- 信息增量应隐含在选题质量和钩子设计中，不需要在正文中对比"别人没写什么"
- 例外：如果中文媒体的错误报道本身就是论点的一部分（如事实错误纠正），可以点名

### "产品含义"过渡约束（更新 2026-05-24）
- "我做产品"这个身份声明每 4 篇最多用 1 次
- 合法替代：直接给出产品设计层面的判断，不需要"我做产品"的身份声明作为过渡——读者已经知道你做产品
- 如果要用个人经验，必须包含具体动作或具体项目，不能只是"我做产品的人觉得…"

### 收尾多样性约束（更新 2026-05-24）
- 同一周内禁止重复使用同一收尾句式模板
- "停止 X。开始 Y。"指令式收尾每 4 篇最多 1 次
- 二元对立问题（"A 还是 B？"）每 3 篇最多 2 次
- 合法替代收尾形态：产物召唤式（给出可复制的 gist / CLAUDE.md snippet）、标签铸造式、纯判断式（不加问题）

### 对标风格
- **Karpathy**：用"I noticed..."起手，个人观察 + 技术洞察，casual 但精确
- **Altman**：极简，一句话一个判断，自信不解释
- **小红书高智感科技内容**：平等对话，有温度但不煽情，像"人间夜市"不像"TED 演讲"

### 时效性锚点
- 每篇内容必须有一个可追溯的时效性事件（KOL 言论、开源项目、行业新闻）
- 开头即锚点，不要写完观点再补事件
- 纯观点没有时效锚点 = 软文，不发

### 结构规则（更新 2026-04-29，从 Pattern Scan W18 提炼追加两种钩子）

**开头四种合法钩子（根据素材决定）：**

| 钩子 | 模式 | 何时用 | 中文版示例句式 |
|---|---|---|---|
| **A. 事件锚点式** | 具体事件 + 真名 + 数字开场 | 突发新闻、硬事实驱动 | "4 月 23 日，Anthropic 在 Forge Global..."（已用过 #004 #005） |
| **B. 钩子结论式** | 一句光秃秃的反常识判断 | 趋势观察、模式识别 | "大多数人都搞错了 ROI 可测性的真实含义。" |
| **C. 反共识自白式** | "我以前以为 X，现在不这么想了" / 圈内人停止主流做法 | 认知翻转、工作流转变（Karpathy 模式）| "我停止用 Claude Code 写代码了。我用它做 [更深层用途]。" |
| **D. 发现式** | "我发现 / 量了 / 挖了 X" + 让读者 30 秒自己验证的精确细节 | 技术细节、数据揭示（Solayer 实习生爆 Claude Code 泄露模式）| "Anthropic 的 [产品] 里藏着 [具体细节]——你 `npm show xxx` 自己看。" |

**🔴 关键洞察（W18 Pattern Scan）**：发现式的护城河**不是观点，是可独立验证的精确细节**。技术精确性创造"自我传播的验证行为"——读者验证完就分享。这一类钩子绕过了粉丝量壁垒，是中小号最强的爆款路径。

**第二段必须是 Complication**：无论 A/B/C/D，第二段都要把冲突点摆出来——"中文报道都在写 X，但 X 错过了关键问题" / "大多数人理解错了" / 一个反预期的转折。**OT-2 案例**：点名"Boris Cherny 出现在 GitHub issue"比"Anthropic 官方回应"冲突感更强——具体人名 > 组织名。

**禁用结构：**
- "三个原因"、"四个维度"这种 MECE 列表式分点（咨询 deck 体，落入 AI 文质感）
- 通篇 BLUF 倒装宣言（"我的结论是 X，因为 A、B、C"——同样咨询体）
- 段间用"接下来聊聊"、"再说一个"、"另外"过渡

**列表的合法用法（W18 边界澄清）：**
- ✅ 列表作为**参考产物**（toolkit / stack 汇总 / 工具合集）——列表本身就是价值，可保留
- ❌ 列表作为**说服线索的中段**——用列表推进论点会弱化单线推进的冲击，禁

**收尾形态（4 种）：**
- 一句有力判断 + 一个真问题（默认）
- **概念提炼式收尾**：把全篇浓缩成一对对立动词指令（"停止把 X 当 Y，开始把它当 Z"），让收尾本身可脱离全文独立流通（Yuchen Jin 蒸馏 Karpathy 模式）
- **召唤式产物收尾**：放出可复制 gist / CLAUDE.md snippet / GitHub link（Karpathy idea file 模式）——适合工作流类内容
- **标签铸造式**：创造一个可传播标签（"AI shrinkflation" 模式），让帖子生命周期靠标签延续

## 句式节奏

- 短句为主，长句点缀
- 三连短句制造节奏感（"没有人推荐。没有投广告。产品团队自己都不知道。"）
- 问句用来推进思考，不用来制造悬念
- 加粗用于核心判断，一篇最多 2-3 处
