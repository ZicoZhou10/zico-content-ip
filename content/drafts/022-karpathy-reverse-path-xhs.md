## 正文

AI 领域最有资格独立的人，正在选择回到实验室。

5 月 19 日，Andrej Karpathy 宣布加入 Anthropic。OpenAI 联合创始人，Tesla Autopilot AI 前负责人，vibe coding 概念的提出者。2024 年离开 OpenAI，创办 Eureka Labs 做教育 AI。AI 顶级研究者的标准路径是"大厂→创业→独立"。Karpathy 走反了——把自己的公司放下，回到别人的实验室。

2025 年 10 月，他在 Dwarkesh Patel 播客上说过一句话：frontier 模型产出的大量 Agent 输出是"slop"。主张要想"十年 Agent"，不要想"一年 Agent"。半年后，他的全职工作变成了用 Agent 做研究。说 Agent 是 slop 的人，现在每天用 Agent 训练下一代 Claude。

他在 Anthropic 的具体职责：加入 pre-training 团队 Nick Joseph 旗下，同时组建一个新小组。任务一句话——**用 Claude 加速产出下一代 Claude 的研究。**

不是用 Claude 写应用。不是用 Claude 做客服。是用 Claude 改进生成 Claude 的训练流程本身。AI 提出代码修改，运行实验，验证改进，丢掉无效的，堆叠有效的。

Karpathy 之前在个人项目 autoresearch 里测过这套方法：AI agent 两天跑约 700 次实验，找到约 20 个可堆叠改进，产出 11% 训练加速。个人笔记本上跑的实验。他加入 Anthropic，是把同样的方法放到 Anthropic 的整个算力集群和研究管线上。规模变了，逻辑没变。

这不是打脸。他看到的是同一个判断的两面。

今天的 Agent 在大多数任务上确实是 slop——格式正确，结论错误，缺乏深度判断。但在一个极特殊的场景——AI 改进自身训练流程——Agent 已经能产出真实、可验证、可堆叠的价值。差别不在 Agent 变强了。在任务选对了。训练实验有明确的验证标准：跑完要么指标涨要么不涨，没有模糊空间。能自我验证的场景，天然过滤掉 slop。

他放下公司回到实验室，还暗示了一个更大的赌注：预训练前沿才是当前最值得投入注意力的地方。产品和创业可以等。模型能力的天花板决定一切下游应用的上限。当 AI 参与训练自己的下一代，天花板的上移速度本身在加速。

**每一轮改进让下一轮来得更快。**

Karpathy 用自己的职业选择回答了一个问题：这个循环是否已经值得放下所有其他事情。他的答案是值得。你的呢？
