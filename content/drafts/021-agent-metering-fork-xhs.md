## 正文

Anthropic 的定价页面本周多了一个新概念：Agent SDK Credits。有人拿典型的 claude -p 工作负载算了一遍——6 月 15 日之后，同样的 Agent 任务，有效价格贵了 12 倍到 175 倍。数学不复杂：Pro 订阅 $20/月，新方案给你 $20 的 Agent Credit，按 API 标准费率扣减。Claude Opus 的 API 价格是 $15/M input + $75/M output。一个中等复杂度的 Agent 任务跑下来几万 token，$20 可能撑不过三天。

这不是"Claude 涨价了"。

过去几个月发生了一件事：有人发现用 $20 Pro 订阅 + OpenClaw 等第三方 Agent 框架，可以跑出等价数百美元 API 调用量的工作负载。Anthropic 先封了第三方访问，社区抗议，现在用 Credit 池方案重新开放。表面是让步。实际是在产品层画了一条正式的分界线：**人用 AI（对话）走订阅，机器用 AI（Agent）走计量。**

两种用量，两种经济模型。

这条线为什么重要？因为它是 Anthropic 在公开承认一个事实：当前 Agent 使用量的规模和波动性，已经超出订阅模型能承载的范围。订阅的前提是用量可预测——每月大致恒定，平台用统计学铺平峰谷。Agent 打破了这个前提。一个程序调 Claude 跑 150 个并行 PR review，用量可以在一小时内波动 100 倍。这不是人类使用软件的模式。

我做产品，对这件事的判断是：Anthropic 不是第一家遇到这个问题的，但它是第一家在产品定价层公开划出人机边界的。OpenAI 的 Codex 目前捆绑在 ChatGPT Pro 里——等 Agent 工作负载继续涨，同样的数学问题会浮出来。Google 的 Gemini API 已经是计量模型，但对话产品还是订阅。所有 AI 平台最终都会走到同一个十字路口：当 AI-to-AI 调用量超过 human-to-AI 调用量，订阅就撑不住了。

6 月 15 日不只是一个计费变更日期。它是 AI 产品从消费级订阅走向工业级计量的第一个公开时间节点。

**如果你在用 Claude 构建 Agent 产品，现在有 29 天重新核算成本结构。之后的问题不再是"选哪个模型"，而是"你的 Agent 每跑一次值多少钱"。**
