## 正文

4 月 21 日，Hugging Face 在 GitHub 开源了一个项目叫 ml-intern。

它做的事：读 arXiv 论文，沿引用图谱找数据集，评估数据质量，写训练脚本，跑训练，看指标，判断方向，再迭代。一个入门级 ML 工程师每天做的事，它一条龙跑完。

官方演示数据：以 Qwen3-1.7B 为起点（GPQA baseline 约 10%），ml-intern 在 10 小时内把成绩推到 32%。对比，Claude Code 在同类任务上得分 22.99%，Codex 在 HealthBench 上也被超过。

中文报道的标题几乎都在写"AI 超过了 Claude Code"。但这个比较框架是错的。

Claude Code 是代码助手——你告诉它写什么，它写。ml-intern 不是代码助手，它在跑一套完整的科研循环：选方向、找资料、设计实验、执行、评估、迭代。比较这两个，就像比较计算器和会计师——一个处理你给的数字，一个决定该看哪些数字。

真正值得关注的不是跑分，是 ml-intern 自动化了什么。

读文献。找数据。写脚本。跑训练。看指标。再来一轮。

这个循环占 ML 工程师日常工作的绝大部分时间。Hugging Face 把它变成了一行 `git clone` 就能启动的东西。

但有一个部分它没有自动化：**哪个方向值得追。**

哪个 benchmark 真正反映实际能力？哪个数据集有隐藏偏差？两个实验结果矛盾的时候该信哪个？这些不是流程问题，是判断力问题。

我给这个现象起了个名字：**执行层坍缩**。

每个职业都有两层。执行层是"按方法做事"——跑实验、写报告、画原型、做竞品分析。判断层是"决定做哪件事"——选方向、定优先级、在不确定中做取舍。

过去两层混在一起，因为执行本身需要大量时间和技能，能执行的人天然获得更多判断机会。ml-intern 做的事，是把 ML 研究的执行层压到接近零成本。当执行接近免费，执行能力就不再稀缺。剩下的稀缺资源只有一个：判断力。

这不只是 ML 工程师的事。

设计师的执行层是出图，判断层是审美方向。律师的执行层是检索案例，判断层是诉讼策略。产品经理的执行层是写文档跑数据，判断层是"这个功能该不该做"。

每个领域的执行层都在被 AI 逐步压平。速度不同，方向一致。

**当执行变得免费，你的职业价值就等于你判断层的厚度。**

一个值得认真想的问题：你现在每天的工作时间里，执行和判断的比例是多少？如果执行层明天归零，剩下的部分撑不撑得起你的岗位？
