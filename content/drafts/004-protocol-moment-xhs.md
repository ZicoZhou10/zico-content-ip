## 正文

4 月 22 日，Google Cloud Next 2026 大会上，Google 公布了一组数据：Agent2Agent（A2A）协议已升级到 v1.2，已有 150 个组织在生产环境、不是测试环境里，通过这套开放协议让不同平台的 AI Agent 互相协作。

同一天，协议治理权交给了 Linux Foundation 旗下的 Agentic AI Foundation。Salesforce、SAP、ServiceNow、微软、AWS 全部接入。Google 同时把 Vertex AI 重组为 Gemini Enterprise Agent Platform，并公布了 7.5 亿美元基金推动咨询公司（麦肯锡、埃森哲等）部署 Agentic AI。

中文报道几乎都在写"Google 向 OpenAI 和 Anthropic 发起挑战"。但竞争不是这件事最重要的部分。**标准化才是。**

A2A 做的事不复杂：定义一套标准，让不同平台、不同公司开发的 Agent 能直接交换任务、共享上下文、完成工作交接。就像 HTTP 让所有浏览器都能访问所有网站——Agent 之间的协作不再需要一对一定制集成。

治理权不在 Google 手里，而是一个开源基金会。这个结构比协议本身更关键：任何单一公司都没法把它关起来。

回看互联网历史：TCP/IP 标准化之前，各家的计算机网络是孤岛。标准化之后，网络效应才真正启动——不是某一家产品变强，而是所有人的产品突然能连在一起。

Agent 世界正在经历同样的转折。

我在产品工作中接触过跨系统 AI 集成，直接感受过那个痛点：让两个不同平台的 Agent 协作，工程复杂度远超预期。数据格式不同、上下文传递方式不同、权限模型不同。每接一个新伙伴，几乎要重写一遍集成层。

A2A 把这件事变成了调一个接口。这个差别不是"更方便"，是"可能"和"不可能"的问题——没有标准协议，跨平台 Agent 协作只有 Salesforce、微软那种体量的公司烧得起；有了标准协议，任何开发者的 Agent 都能加入网络。

**Agent 的网络效应，从 4 月 22 日这天开始具备条件了。**

Draft #001 写过 Agent 正在变成"用户"——自己找到产品，自己接入，自己消费服务。那是 Agent 经济的需求侧。

A2A 补上的是供给侧的基础设施：Agent 之间的分工、交易、协作有了标准化通道。需求侧和供给侧都到位，Agent 经济从概念进入基建阶段。

我给这个转折起了个名字：**协议时刻**。不是某个产品变强，而是整个生态突然获得了互联互通的能力。互联网的协议时刻是 1983 年 TCP/IP 成为 ARPANET 标准。移动互联网的协议时刻是 HTTP 加 App Store 的标准化分发。Agent 互联网的协议时刻，是 A2A v1.2 进入生产部署的这一天。

一个值得想的问题：当 Agent 可以低摩擦跨 Salesforce、SAP、微软、AWS 协作，"平台"的护城河变成什么？过去平台的壁垒是锁住用户和数据。如果 Agent 自由穿梭于平台之间，用户留在哪里不再由平台决定，而由 Agent 的路由逻辑决定。

**平台锁定用户的时代之后，谁来锁定 Agent？还是根本没人能锁定？**
