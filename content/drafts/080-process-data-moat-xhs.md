## 正文

我以为 AI 编程模型训练数据的壁垒是代码仓库规模——谁爬了更多 GitHub 仓库、谁清洗得更干净，谁的编程能力就更强。7 月 8 日看完 xAI 发布 Grok 4.5 的训练方式之后，我修正了这个判断。

问题在于：过去所有前沿 AI 编程模型——Claude、GPT、Gemini——训练用的都是代码仓库快照。快照是结果，不是过程。Grok 4.5 训练用的是另一种东西：Cursor 上百万开发者每天写代码时产生的完整行为轨迹。一个开发者怎么开始写一个功能、中途怎么放弃一种写法换另一种、怎么在报错后定位 bug、怎么重构一段不满意的代码。代码仓库告诉模型"好代码长什么样"。行为轨迹告诉模型"好代码是怎么一步步写出来的"。

交易结构很清楚。xAI 有 1.5 万亿参数的 V9 架构和大规模 GPU 集群。Cursor 有数百万月活开发者每天产生的行为数据。xAI 用算力换 Cursor 的行为轨迹，Cursor 用数据换一个在自己工具里最好用的专属模型。Grok 4.5 上线当天直接进入 Cursor 全部订阅方案。500K token 上下文窗口，API 定价 $2/$6 per million token。

这笔交易创造了一个闭环飞轮：开发者用 Cursor → Cursor 收集行为轨迹 → 轨迹训练 Grok → Grok 成为 Cursor 里最好用的模型 → 更多开发者涌入 Cursor → 更多行为轨迹。**飞轮每转一圈，数据更多，模型更强，用户更多。没有第三方能插进来。**

Anthropic 和 OpenAI 也在 Cursor 里分发自己的模型。但它们拿不到 Cursor 的行为轨迹数据——它们是分发渠道里的租客。xAI 通过联合训练协议拿到了数据权限，变成了房东。同一栋楼里，租客以为自己在做生意，房东在积累资产。定价让竞争压力更直接：Grok 4.5 主张 Opus 4.8 级别性能，平均 token 消耗只有 Opus 的四分之一，Arena.ai 前端代码评分排名第一。对一个每天在 Cursor 里写代码的开发者来说，这是算术题，不是品牌忠诚题。

数据飞轮在 AI 行业讲了两年，大部分时候停在幻灯片上。Grok 4.5 是第一个完整跑通"AI 工具使用数据 → 训练下一代模型"闭环的实战案例。壁垒不在代码仓库——那个人人能爬。壁垒在**过程数据**：用户在你的工具里做决策的完整行为序列，只有工具拥有者才能收集。

**代码仓库人人能爬。行为轨迹只有工具的拥有者才能收集。AI 模型竞争的下一道壁垒，已经从"谁的算力大"转移到"谁的工具里有更多人在真实工作"。**
