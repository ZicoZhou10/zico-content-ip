## 正文

7 月 16 日，Moonshot AI 直接上线 Kimi K3。2.8 万亿参数，Stable LatentMoE 架构，896 个专家每次推理激活 16 个。两个版本同步发布：K3 Max 跑日常 Agent 任务，K3 Swarm Max 专为大规模并发设计——单次任务可扇出数百个并发子 Agent。API 定价 $3/$15 per million token。开放权重承诺 7 月 27 日发布。

第二天，Axios 标题：「China just erased America's AI lead」。Fortune：出现「DeepSeek 闪回」。美国科技股盘中下跌。

闪回是准确的。但不是因为基准测试——K3 自报超过 Opus 4.8 这件事本身不意外，过去一年几乎每隔几周就有一个新模型声称超过前一个。市场真正在意的是另一个东西。2025 年 1 月 DeepSeek 打穿的假设是「高能力模型必须高成本」，同等能力的 per-token 价格下降 90%+。2026 年 7 月 Kimi K3 正在打穿的假设是下一层：**大规模 Agent 并发必须高成本。** 两次结构相同——一个成本假设崩塌，一类此前不可行的应用变得可行。

Agent 群的推理成本随并发数线性增长。当前沿模型 output 定价在 $10–15/M token 量级时，扇出数百个子 Agent 各执行多步推理，单次运行的成本快速堆积到数百甚至数千美元。企业场景下这类任务不是跑一次——是每天跑、反复跑。「Agent 经济」听上去很好，一算账就变成「Agent 成本中心」。能承受这个支出的团队名单很短。

K3 Swarm Max 的差异不在参数量大，在架构为并发而生。MoE 稀疏激活：2.8 万亿参数每次只启动 16 个专家，实际计算密度远低于同参数密集模型。数百子 Agent 扇出时的边际成本和密集模型不在同一条曲线上。开源权重允许自部署，进一步压缩边际成本。三层叠加的效果：大规模 Agent 并发正在从「只有最有钱的公司跑得起」向「中等团队可以认真评估」移动。

Agent 的单体能力早就不是瓶颈。SemiAnalysis 刚报告 Claude Code 贡献了全球 7%+ 的 GitHub 提交——单个 Agent 做生产级工作已被验证。真正的门槛在规模端：需要数百个 Agent 同时执行时，成本结构决定了它是白皮书里的概念还是损益表上的选项。

**K3 的意义不是「中国模型追上了美国」。是 Agent 并发运行的经济可行线又往下移了一格。** DeepSeek 让单次模型调用变便宜。K3 Swarm Max 正在让 Agent 群运行变便宜。当这条成本线降到足够低，第一批真正规模化的 Agent 应用会在哪个行业先跑通？
