## 正文

过去一年，我默认了一个前提：AI Agent 是开发者的工具。Claude Code、Cursor、Codex——整个赛道的命名、融资、benchmark 都围着代码转。

7 月 7 日，Anthropic 公布了 Claude Cowork 的真实使用数据，同时宣布 Cowork 从桌面端扩展到 web、iOS 和 Android。8.7%。所有 Cowork 会话中，只有 8.7% 和软件开发有关。最大单一用途：业务流程与运营，33.4%——是代码的四倍。内容创作与文案 16.4%。两项加起来超过一半的使用量，和写代码没有关系。

Cowork 是和 Claude Code 同期发布的产品——定位靠近开发者生态，发布节奏跟着 AI coding agent 竞争周期走。但用户完全不顾产品定位。他们把一个开发者生态的工具做成了通用知识工作的委托层。写周报、整理流程文档、追踪项目进度、起草文案——这些任务不需要 frontier 模型能力，但一直没人接走。Cowork 上线，它们第一个被填上。

更值得注意的是 Anthropic 这次发布的功能方向。会话跨设备同步。用户不在线时 Claude 继续在后台执行。遇到需要判断的节点，主动向用户手机推送决策请求。这不是"AI 助手等你下指令"——是"AI 在干活，需要你拍板的时候叫你一声"。工作关系倒过来了。

同一周，Anthropic 在 Slack 里上线了 Claude Tag——团队级多人协作 AI Agent。不是一个人给 AI 布置任务，是多个人和 AI 在同一个 Slack 频道里协作。Cowork 管个人委托，Tag 管团队协作。两条线同步推的含义只有一个：Anthropic 正在从开发者工具公司变成组织基础设施公司。

TechCrunch 的标题写了：《The coding agent wars are spilling into the rest of the office》。方向反了。不是编程工具在"溢出"到办公场景——是办公场景的需求从一开始就比代码需求大得多，只是没人量过。

8.7% 暴露的不只是一个产品定位偏差。是整个行业持续两年的前提错误。把"AI Agent"和"AI coding"画了等号。用 coding benchmark 衡量进展，用写代码能力选工具，用 coding agent 讲融资故事。但大多数知识工作者的高频痛点从来不是写代码——是撰写、整理、分析、跟进、执行。

**AI Agent 的扩散路径不是"先服务开发者，再推广到其他人"。Cowork 的数据说了另一件事：被开发者发现的工具，立刻就被全部知识工作者接管了。没有过渡期。**

围绕"AI Agent = 开发者工具"建立的竞争策略、产品定位、投资叙事，前提错了。下一轮竞争不在谁的代码写得好——**在谁先拿下知识工作的委托入口。**
