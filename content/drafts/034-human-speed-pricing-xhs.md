## 正文

你为 AI 编程工具付的月费，不是按你的使用量定的——是按"一个人类使用 AI 的速度"定的。

6 月 1 日，GitHub Copilot 正式从席位计费切换到 token 计量。新机制：代码补全和行内建议维持无限额，其余所有功能——Agent 任务、AI 聊天、多文件编辑——按 token 消耗扣 AI Credits。Pro 月费 $10，包含 $10 credits（1 credit = $0.01）。Pro+ 月费 $39，包含 $39 credits。听起来一一对应。

TechCrunch 5 月 30 日报道的一个数字撕开了这层对应：一个 agentic 编程会话——让 Copilot agent 重构一个遗留模块——消耗 $30–40 的 credits。一次会话。整月额度的三到四倍。

GitHub 社区讨论帖积累了 900 多个踩、400 多条评论。Visual Studio Magazine 引用开发者原话："PRU 和 token 计费之间的差距能让喝醉酒的人因震惊而清醒。"

开发者在骂价格。但 GitHub 遇到的问题比价格深一层。

Copilot Pro $10/月的定价，逻辑前提是：一个开发者每月发起多少次 AI 请求，有自然天花板。人写代码，人读建议，人决定采不采纳——每个环节都有人的节奏在控制消耗速度。代码补全时代这个模型没有问题。

Agent 模式打破了前提。一个 agentic 编程任务不等人确认就并行展开：读文件、改代码、跑测试、读报错、再改。循环运行，不停不等。单次会话的 token 消耗和人手动对话 AI 不在一个数量级。

GitHub 不是定价定错了。GitHub 是发现自己的定价架构建在"用户以人的速度使用 AI"这个假设上。Agent 不以人的速度运行。代码补全保持免费——那是人在用。Agent 任务按 token 收费——那是机器在用。GitHub 把"人用"和"Agent 用"拆开定价了，因为它们从来不是同一种使用行为。

企业端的同一面墙三天前已经撞过。Uber 四个月烧完全年 $34 亿 AI 预算，一家匿名企业 30 天花了 $5 亿 Claude 账单。那是需求端——企业没预料到 Agent 的消耗速度。GitHub 是供给端——它是第一个公开承认定价模型不兼容 Agent 时代、并主动重建架构的主流开发者工具。

**GitHub 不是在涨价。它是在承认一件事：按月定价的逻辑假设用户是人。Agent 不是人。你付的月费买的不是 AI 能力——是"你使用 AI 的速度"。当速度不再是你的，价格就不再是你的。**

Copilot 是第一个公开重建的。Claude、Cursor、Windsurf 全在同一条路上。**下一个谁动？谁在假装这面墙不存在？**
