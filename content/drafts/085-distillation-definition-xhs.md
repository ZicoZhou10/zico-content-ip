## 正文

蒸馏不是偷窃。用大模型的输出训练小模型，Hinton 1987 年就在做。整个 ML 行业对知识蒸馏不陌生。**但 7 月 22 日，白宫说它是。**

当天，白宫科技政策办公室主任 Michael Kratsios 公开指名 Moonshot AI，指控其通过「大规模工业级蒸馏」从 Anthropic Fable 5 的 API 输出中提取能力，训练出 Kimi K3。措辞不是学术讨论——是「知识产权盗窃」。同日，财政部长 Scott Bessent 跟进：如果工业级蒸馏认定构成 IP 盗窃，美国可能对 Moonshot AI 实施制裁或列入 Entity List。Kratsios 同时指控 Moonshot 通过泰国获取 Nvidia GB300 算力，可能违反出口管制。

这里有一个被跳过的事实。Fable 5 的 API 是付费开放的。任何企业都可以调用，拿到输出，用于自己的产品和研究。Anthropic 的服务条款确实禁止「利用输出训练竞争模型」——但这是合同条款，不是法律定义。合同违约和「知识产权盗窃」之间的距离，被白宫一步跨过了。

时间线本身也经不起推敲。Fable 5 API 7 月 1 日才开放。Kimi K3 7 月 16 日发布。15 天。多名 ML 研究员公开质疑：2.8T 参数的模型不可能在 15 天内主要靠 API 蒸馏完成训练。Kratsios 的指控是政治声明，不是技术报告。

更深的矛盾在这里。OpenAI 和 Anthropic 在训练自己的大模型时，用了互联网上数十亿网页、数百万书籍——大多数在没有明确授权的情况下抓取。OpenAI 正在被 New York Times 起诉，用的正是「你在没有授权的情况下学习了我们的内容」这个逻辑。**如果「从他人内容的输出中学习」是盗窃，这个标准往回套，美国 AI 公司自己首先过不了。**

但这不是法律一致性问题。这是**定义权**问题。

谁有权定义「蒸馏何时构成盗窃」？不是法院——没有判例。不是 ML 社区——没有共识。是白宫。是出口管制机构。是可以签制裁令的行政权力。AI 知识产权的边界，正在被地缘政治工具而不是版权法来划定。

下一步的逻辑已经清晰。美国 AI 实验室会被施压收紧 API 访问——加地理限制、用途审查、实时监控。不是因为技术上有这个必要，是因为政策需要一个执法抓手。API 开放性是过去两年 AI 生态的基础假设。这个假设正在被重新定价。

**蒸馏是 40 年的标准技术。但「谁可以蒸馏谁」，从 7 月 22 日起，是地缘政治问题。**
