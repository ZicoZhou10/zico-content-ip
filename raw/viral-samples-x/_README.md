# X 爆款样本 · 手动喂养工作流（可选）

**目的**：你平时刷 X 时看到惊艳的推文，扔进来。Weekly Pattern Scan trigger 会读这个文件夹的本周样本，和它自己 WebSearch 的结果合并提炼。

**核心价值**：你刷 X 时的品味过滤是世界上最稀缺的信号源——比任何爬虫、API、新闻聚合都准。Pattern Scanner 自己 WebSearch 抓不到的中小号 outlier，靠你这条人工通道补齐。

---

## 何时用 / 不用

**用**：刷到一条让你愣 2 秒的推文，无论作者粉丝多少。重点关注：
- 中小号（< 50K 粉）单帖互动 ≥ 5K
- 一句话钩子特别狠的（让你想保存或复用的句式）
- 一个反共识自白 / 发现 / 概念提炼让你觉得"这个写法可以套用到中文"

**不用**：
- 单纯转发新闻（除非转发文案本身写得好）
- 头部 KOL 的常规发言（结构信号被粉丝量污染）
- 你已经在 viral-patterns.md 里见过的同类样本

---

## 工作流（每条 30 秒）

1. 在 X 上看到一条想留下来的推文
2. 复制推文链接（点 share → copy link）
3. 在 Obsidian 打开本周对应的 `2026-WXX.md` 文件
4. 粘贴链接 + 一行你的直觉判断（"这条钩子是发现式但走在大众预期前"）
5. 保存。Obsidian Git 自动 push。

不用复制推文内容——Pattern Scanner 会自己访问链接（如果可达）；如果链接被付费墙挡，你的一句话直觉就是它能用的全部信号，照样能提炼。

---

## 格式

```markdown
---
url: https://x.com/handle/status/xxxxx
likes: <估计或精确数字，可空>
follower_tier: small / medium / large
my_take: <你的一句话直觉判断，关键>
---
```

样本之间用 `---` 单独一行分隔。
